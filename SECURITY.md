## Reporting a Vulnerability

We take the security of our software seriously. If you believe you have found a security vulnerability in the Network Test Tool, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to [your-email@example.com]. You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

Please include the following details in your report:

*   Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
*   Full paths of source file(s) related to the manifestation of the issue
*   The location of the affected source code (tag/branch/commit or direct URL)
*   Any special configuration required to reproduce the issue
*   Step-by-step instructions to reproduce the issue
*   Proof-of-concept or exploit code (if possible)
*   Impact of the issue, including how an attacker might exploit the issue

We prefer all communications to be in English.

## Policy

*   We will investigate and respond to all valid reports.
*   We will do our best to patch vulnerabilities within 90 days.
*   We will acknowledge your contribution in our security advisories (if you wish).
